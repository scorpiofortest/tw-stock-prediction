## 1. 九大訊號加權評分系統

### 總覽

每個訊號獨立計算一個分數，範圍為 **-100 ~ +100**（正值看漲、負值看跌、0 為中性）。最終加權總分同樣落在 -100 ~ +100。

| # | 訊號名稱 | 權重 | 類別 |
|---|---------|------|------|
| 1 | 外盤比率 | 15% | 量能方向 |
| 2 | 五檔委買 vs 委賣壓力 | 12% | 掛單意圖 |
| 3 | 最近10筆成交明細方向 | 10% | 短線動量 |
| 4 | 日內高低位置 | 8% | 價格結構 |
| 5 | 即時漲跌幅動能 | 10% | 動能趨勢 |
| 6 | RSI | 12% | 技術指標 |
| 7 | MACD OSC 正負 | 13% | 趨勢確認 |
| 8 | KD 黃金/死亡交叉 | 12% | 交叉訊號 |
| 9 | 盤中走勢加速度 | 8% | 加速度 |
| | **合計** | **100%** | |

**權重設計理念**：
- 量能方向類（外盤比率 15%）與趨勢確認類（MACD 13%）權重最高，因為成交量方向與趨勢是盤中最可靠的短線指標
- 掛單意圖（五檔 12%）和技術指標（RSI 12%、KD 12%）為次要確認
- 短線動量（10筆成交 10%、漲跌幅動能 10%）提供即時脈動
- 價格結構（日內位置 8%）和加速度（8%）為輔助修正

### 1.1 外盤比率（權重：15%）

**定義**：在成交明細中，以「外盤」（以委賣價成交）佔總成交量的百分比來判斷買方主動攻擊的強弱。

**資料來源**：當日累積成交明細的內外盤統計

**計算公式**：

```
外盤比率 = 外盤成交量 / (外盤成交量 + 內盤成交量) × 100%
```

**評分轉換**：

```
if 外盤比率 >= 75%:
    score = +80 + (外盤比率 - 75) × 0.8          # 最高 +100
elif 外盤比率 >= 60%:
    score = +30 + (外盤比率 - 60) × (50 / 15)    # +30 ~ +80
elif 外盤比率 >= 55%:
    score = +10 + (外盤比率 - 55) × (20 / 5)     # +10 ~ +30
elif 外盤比率 >= 45%:
    score = (外盤比率 - 50) × 2                   # -10 ~ +10（中性區）
elif 外盤比率 >= 40%:
    score = -10 - (45 - 外盤比率) × (20 / 5)     # -10 ~ -30
elif 外盤比率 >= 25%:
    score = -30 - (40 - 外盤比率) × (50 / 15)    # -30 ~ -80
else:
    score = -80 - (25 - 外盤比率) × 0.8           # 最低 -100
```

**實例**：外盤比率 = 62% → score = +30 + (62-60) × 3.33 = +36.67 → 取整 +37

### 1.2 五檔委買 vs 委賣壓力（權重：12%）

**定義**：比較五檔委買總量與五檔委賣總量的比例，判斷市場掛單意圖的偏多或偏空。

**資料來源**：即時五檔報價（bid1~bid5 的量、ask1~ask5 的量）

**計算公式**：

```
委買總量 = sum(bid_volume[1..5])
委賣總量 = sum(ask_volume[1..5])
壓力比 = (委買總量 - 委賣總量) / (委買總量 + 委賣總量)
# 壓力比 範圍：-1.0 ~ +1.0
```

**評分轉換**：

```
if 壓力比 >= 0.5:
    score = +80 + (壓力比 - 0.5) × 40             # +80 ~ +100
elif 壓力比 >= 0.2:
    score = +30 + (壓力比 - 0.2) × (50 / 0.3)     # +30 ~ +80
elif 壓力比 >= 0.05:
    score = +5 + (壓力比 - 0.05) × (25 / 0.15)    # +5 ~ +30
elif 壓力比 >= -0.05:
    score = 壓力比 × 100                           # -5 ~ +5（中性區）
elif 壓力比 >= -0.2:
    score = -5 - (|壓力比| - 0.05) × (25 / 0.15)  # -5 ~ -30
elif 壓力比 >= -0.5:
    score = -30 - (|壓力比| - 0.2) × (50 / 0.3)   # -30 ~ -80
else:
    score = -80 - (|壓力比| - 0.5) × 40            # -80 ~ -100
```

**注意事項**：
- 五檔掛單容易被「掛假單」操控，需結合成交明細交叉驗證
- 當委買/委賣總量任一方 < 50張時，可信度降低，建議將此訊號權重減半
- 股價在漲停/跌停附近時，掛單結構會嚴重失真，應忽略此訊號

### 1.3 最近10筆成交明細方向（權重：10%）

**定義**：觀察最近10筆逐筆成交明細，統計外盤筆數與內盤筆數的比例，反映最短期的主動攻擊方向。

**資料來源**：逐筆成交明細（tick data），需區分每筆是外盤或內盤

**計算公式**：

```
外盤筆數 = count(最近10筆中以委賣價成交的筆數)
內盤筆數 = 10 - 外盤筆數
方向比 = (外盤筆數 - 內盤筆數) / 10
# 方向比 範圍：-1.0 ~ +1.0（步進 0.2）
```

**加權改良**（考慮成交量大小）：

```
加權外盤量 = sum(每筆外盤成交的張數)
加權內盤量 = sum(每筆內盤成交的張數)
加權方向比 = (加權外盤量 - 加權內盤量) / (加權外盤量 + 加權內盤量)
```

**最終方向值**：取簡單方向比(40%)與加權方向比(60%)的混合

```
最終方向值 = 0.4 × 方向比 + 0.6 × 加權方向比
```

**評分轉換**：

```
score = clamp(最終方向值 × 100, -100, +100)
```

**實例**：10筆中 7 筆外盤、3 筆內盤，且外盤的平均成交量較大：
- 方向比 = (7-3)/10 = 0.4
- 假設加權方向比 = 0.55
- 最終方向值 = 0.4×0.4 + 0.6×0.55 = 0.49
- score = +49

### 1.4 日內高低位置（權重：8%）

**定義**：目前即時價格位於今日最高價與最低價之間的相對位置。

**資料來源**：當日最高價、最低價、目前價

**計算公式**：

```
if 最高價 == 最低價:
    位置百分比 = 50%                  # 無振幅時視為中間位置
else:
    位置百分比 = (目前價 - 最低價) / (最高價 - 最低價) × 100%
```

**評分轉換**：

```
# 非線性轉換：位置越極端，分數變化越大
if 位置百分比 >= 90%:
    score = +70 + (位置百分比 - 90) × 3            # +70 ~ +100（強勢高檔）
elif 位置百分比 >= 70%:
    score = +20 + (位置百分比 - 70) × 2.5          # +20 ~ +70（偏多）
elif 位置百分比 >= 50%:
    score = (位置百分比 - 50) × 1                  # 0 ~ +20（略偏多）
elif 位置百分比 >= 30%:
    score = (位置百分比 - 50) × 1                  # -20 ~ 0（略偏空）
elif 位置百分比 >= 10%:
    score = -20 - (30 - 位置百分比) × 2.5          # -20 ~ -70（偏空）
else:
    score = -70 - (10 - 位置百分比) × 3            # -70 ~ -100（弱勢低檔）
```

**進階修正**：結合日內振幅

```
振幅 = (最高價 - 最低價) / 開盤價 × 100%
if 振幅 < 0.3%:
    # 振幅太小，位置資訊不可靠，衰減分數
    score = score × 0.3
elif 振幅 < 0.8%:
    score = score × 0.6
# 振幅 >= 0.8% 時維持原始分數
```

### 1.5 即時漲跌幅動能（權重：10%）

**定義**：以目前即時漲跌幅及其與前一分鐘漲跌幅的變化量，評估價格動能強度與方向。

**資料來源**：即時成交價、昨收價、前1分鐘的漲跌幅紀錄

**計算公式**：

```
漲跌幅 = (目前價 - 昨收價) / 昨收價 × 100%
前一分鐘漲跌幅 = (前一分鐘價 - 昨收價) / 昨收價 × 100%
動能變化 = 漲跌幅 - 前一分鐘漲跌幅                 # 每分鐘的漲跌幅變化率
```

**評分公式**：結合「當前漲跌幅位置」與「動能變化方向」

```
# 基礎分：漲跌幅的絕對位置
if 漲跌幅 >= 5%:
    基礎分 = +80
elif 漲跌幅 >= 2%:
    基礎分 = +40 + (漲跌幅 - 2) × (40 / 3)
elif 漲跌幅 >= 0.5%:
    基礎分 = +10 + (漲跌幅 - 0.5) × (30 / 1.5)
elif 漲跌幅 >= -0.5%:
    基礎分 = 漲跌幅 × 20                          # -10 ~ +10
elif 漲跌幅 >= -2%:
    基礎分 = -10 - (|漲跌幅| - 0.5) × (30 / 1.5)
elif 漲跌幅 >= -5%:
    基礎分 = -40 - (|漲跌幅| - 2) × (40 / 3)
else:
    基礎分 = -80

# 動能修正：動能變化影響 ±20 分
動能分 = clamp(動能變化 × 100, -20, +20)

score = clamp(基礎分 + 動能分, -100, +100)
```

**實例**：漲跌幅 = +1.2%，前一分鐘 = +0.9%
- 基礎分 = +10 + (1.2 - 0.5) × 20 = +24
- 動能變化 = 1.2 - 0.9 = 0.3 → 動能分 = +20（上限）
- score = +44

### 1.6 RSI（權重：12%）

**定義**：使用 14 期 RSI 判斷超買超賣，並結合**反轉邏輯**——RSI 極端值代表的是反轉風險而非趨勢延續。

**資料來源**：盤中 1 分 K 線收盤價計算 14 期 RSI

**RSI 計算公式**（標準 Wilder RSI）：

```
RS = 平均漲幅(14期) / 平均跌幅(14期)
RSI = 100 - (100 / (1 + RS))
```

**評分轉換（含反轉邏輯）**：

```
if RSI > 80:
    # 超買區：反轉做空訊號
    score = -50 - (RSI - 80) × 2.5                # -50 ~ -100
    # 說明：RSI > 80 → 過熱，預期回檔
elif RSI > 65:
    # 偏多區：趨勢向上但接近過熱
    score = +30 + (RSI - 65) × (20 / 15)          # +30 ~ +50
elif RSI > 55:
    # 溫和多方
    score = +10 + (RSI - 55) × 2                  # +10 ~ +30
elif RSI >= 45:
    # 中性區
    score = (RSI - 50) × 2                        # -10 ~ +10
elif RSI >= 35:
    # 溫和空方
    score = -10 - (45 - RSI) × 2                  # -10 ~ -30
elif RSI >= 20:
    # 偏空區
    score = -30 - (35 - RSI) × (20 / 15)          # -30 ~ -50
else:
    # 超賣區：反轉做多訊號
    score = +50 + (20 - RSI) × 2.5                # +50 ~ +100
    # 說明：RSI < 20 → 過冷，預期反彈
```

**反轉邏輯說明**：
- RSI > 80 時，評分為**負值**（預期回檔），而非繼續看漲
- RSI < 20 時，評分為**正值**（預期反彈），而非繼續看跌
- 這符合均值回歸的短線交易邏輯，適用於盤中 62 秒驗證週期
- 注意：在極端趨勢行情中，RSI 可能長期維持在超買/超賣區，此時應降低此訊號的有效權重（見 §2.3 衝突處理）

### 1.7 MACD OSC 正負（權重：13%）

**定義**：使用 MACD 柱狀體（OSC = DIF - MACD signal）的值與方向變化來判斷趨勢動能。

**資料來源**：盤中 1 分 K 線收盤價計算 MACD（12, 26, 9）

**MACD 計算公式**：

```
EMA12 = EMA(close, 12)
EMA26 = EMA(close, 26)
DIF = EMA12 - EMA26
Signal = EMA(DIF, 9)
OSC = DIF - Signal                                # 柱狀體
```

**評分轉換**：

```
# 將 OSC 正規化：以最近 60 根 K 棒的 OSC 絕對值最大值作為基準
max_osc = max(|OSC| over last 60 bars)
if max_osc == 0:
    normalized_osc = 0
else:
    normalized_osc = OSC / max_osc                # 範圍約 -1.0 ~ +1.0

# 基礎分
基礎分 = normalized_osc × 70                      # -70 ~ +70

# 方向修正：OSC 是否在增大或縮小
osc_delta = OSC[current] - OSC[previous]
if osc_delta > 0 and OSC > 0:
    方向修正 = +15                                 # 多頭加速
elif osc_delta < 0 and OSC < 0:
    方向修正 = -15                                 # 空頭加速
elif osc_delta > 0 and OSC < 0:
    方向修正 = +10                                 # 空頭收斂（可能反轉向上）
elif osc_delta < 0 and OSC > 0:
    方向修正 = -10                                 # 多頭收斂（可能反轉向下）
else:
    方向修正 = 0

# 零軸穿越加分
if OSC[previous] < 0 and OSC[current] > 0:
    零軸加分 = +15                                 # 由負轉正（多頭訊號）
elif OSC[previous] > 0 and OSC[current] < 0:
    零軸加分 = -15                                 # 由正轉負（空頭訊號）
else:
    零軸加分 = 0

score = clamp(基礎分 + 方向修正 + 零軸加分, -100, +100)
```

### 1.8 KD 黃金/死亡交叉（權重：12%）

**定義**：使用隨機指標 KD（9, 3, 3），以 K 值與 D 值的交叉以及當前位置來判斷短線買賣訊號。

**資料來源**：盤中 1 分 K 線的 KD 指標

**KD 計算公式**：

```
RSV = (close - lowest_low(9)) / (highest_high(9) - lowest_low(9)) × 100
K = K_prev × (2/3) + RSV × (1/3)
D = D_prev × (2/3) + K × (1/3)
```

**交叉狀態判斷**：

```
golden_cross = (K[prev] < D[prev]) and (K[curr] > D[curr])   # 黃金交叉
death_cross  = (K[prev] > D[prev]) and (K[curr] < D[curr])   # 死亡交叉
k_above_d    = K[curr] > D[curr]                               # K在D上方
k_below_d    = K[curr] < D[curr]                               # K在D下方
```

**評分轉換**：

```
# 1. 交叉事件分（發生在當根 K 棒）
if golden_cross:
    交叉分 = +50
elif death_cross:
    交叉分 = -50
elif k_above_d:
    交叉分 = +15                                   # K在D上方，多頭持續
elif k_below_d:
    交叉分 = -15                                   # K在D下方，空頭持續
else:
    交叉分 = 0

# 2. 位置加權（低檔黃金交叉 / 高檔死亡交叉更有效）
if golden_cross and K[curr] < 30:
    位置加分 = +30                                  # 低檔黃金交叉（強烈買入）
elif golden_cross and K[curr] < 50:
    位置加分 = +15                                  # 中低檔黃金交叉
elif death_cross and K[curr] > 70:
    位置加分 = -30                                  # 高檔死亡交叉（強烈賣出）
elif death_cross and K[curr] > 50:
    位置加分 = -15                                  # 中高檔死亡交叉
else:
    位置加分 = 0

# 3. K值趨勢分
k_trend = K[curr] - K[prev]
趨勢分 = clamp(k_trend × 2, -20, +20)

score = clamp(交叉分 + 位置加分 + 趨勢分, -100, +100)
```

**實例**：K 從 25 穿越 D 向上（低檔黃金交叉），K 趨勢 = +5
- 交叉分 = +50
- 位置加分 = +30（K < 30，低檔）
- 趨勢分 = +10
- score = +90（強烈看漲訊號）

### 1.9 盤中走勢加速度（權重：8%）

**定義**：計算價格變化的「加速度」，即速度的變化率。正加速度代表上漲加速或下跌減速；負加速度代表下跌加速或上漲減速。

**資料來源**：最近 3 分鐘的成交價

**計算公式**：

```
# 速度（每分鐘價格變化率）
v1 = (price[t-1] - price[t-2]) / price[t-2] × 100%    # t-2 到 t-1 的速度
v2 = (price[t]   - price[t-1]) / price[t-1] × 100%    # t-1 到 t 的速度

# 加速度
acceleration = v2 - v1                                  # 速度的變化量（%）
```

**評分轉換**：

```
# 正規化：台股個股盤中每分鐘價格變化通常在 ±0.3% 內
# 加速度通常在 ±0.2% 內
normalized_acc = clamp(acceleration / 0.2, -1.0, +1.0)

# 基礎分
基礎分 = normalized_acc × 60                           # -60 ~ +60

# 方向一致性加分：加速度方向與速度方向一致時加分
if v2 > 0 and acceleration > 0:
    一致性加分 = +20                                    # 上漲加速
elif v2 < 0 and acceleration < 0:
    一致性加分 = -20                                    # 下跌加速
elif v2 > 0 and acceleration < 0:
    一致性加分 = -10                                    # 上漲減速（可能反轉）
elif v2 < 0 and acceleration > 0:
    一致性加分 = +10                                    # 下跌減速（可能反彈）
else:
    一致性加分 = 0

score = clamp(基礎分 + 一致性加分, -100, +100)
```

**注意事項**：
- 開盤前 5 分鐘（09:00~09:05）價格劇烈波動，加速度失真，建議將此訊號權重降低至 50%
- 午盤（11:30~12:30）成交量萎縮，加速度容易被單筆大單扭曲，可信度降低

## 2. 綜合判斷邏輯

### 2.1 加權總分計算

```
加權總分 = Σ(訊號i的分數 × 訊號i的權重)

= score_外盤比率 × 0.15
+ score_五檔壓力 × 0.12
+ score_10筆成交 × 0.10
+ score_日內位置 × 0.08
+ score_漲跌幅動能 × 0.10
+ score_RSI × 0.12
+ score_MACD_OSC × 0.13
+ score_KD交叉 × 0.12
+ score_加速度 × 0.08
```

### 2.2 判斷閾值

| 加權總分範圍 | 判斷 | 預測方向 |
|-------------|------|---------|
| +40 ~ +100 | **強烈看漲** | 62秒後上漲 |
| +20 ~ +39 | **看漲** | 62秒後可能上漲 |
| +5 ~ +19 | **微幅看漲** | 62秒後略偏上漲 |
| -4 ~ +4 | **中性** | 62秒後方向不明 |
| -19 ~ -5 | **微幅看跌** | 62秒後略偏下跌 |
| -39 ~ -20 | **看跌** | 62秒後可能下跌 |
| -100 ~ -40 | **強烈看跌** | 62秒後下跌 |

### 2.3 信心值計算

信心值反映的是「各訊號之間的一致性」而非單純的總分大小。

**計算公式**：

```
# 1. 方向一致性
同方向訊號數 = count(訊號分數與加權總分同號的訊號)
方向一致性 = 同方向訊號數 / 9 × 100%

# 2. 強度因子
強度因子 = |加權總分| / 100 × 100%

# 3. 離散度（分數越集中，信心越高）
分數標準差 = std(所有9個訊號的原始分數)
離散度因子 = max(0, 1 - 分數標準差 / 80) × 100%

# 最終信心值
信心值 = 方向一致性 × 0.45 + 強度因子 × 0.30 + 離散度因子 × 0.25

# 限制範圍
信心值 = clamp(信心值, 10%, 95%)
```

**信心值分級**：

| 信心值範圍 | 含義 |
|-----------|------|
| 80% ~ 95% | 高度確信，訊號高度一致 |
| 60% ~ 79% | 中度確信，多數訊號一致 |
| 40% ~ 59% | 低度確信，訊號分歧 |
| 10% ~ 39% | 極低確信，訊號矛盾嚴重 |

### 2.4 訊號衝突處理

當關鍵訊號之間出現矛盾時，需要特殊處理：

**衝突場景 1：量能方向 vs 技術指標矛盾**

```
# 外盤比率強烈看漲 (+60以上) 但 RSI 超買反轉 (-50以下)
if score_外盤比率 > 60 and score_RSI < -50:
    # 短線量能主導，降低 RSI 反轉權重
    effective_weight_RSI = 0.06         # 從 12% 降至 6%
    # 多出的 6% 分配給外盤比率
    effective_weight_外盤比率 = 0.21     # 從 15% 升至 21%
```

**衝突場景 2：KD 黃金交叉 vs MACD 死亡交叉**

```
# 兩個交叉訊號矛盾
if score_KD > 40 and score_MACD < -40:
    # 取時間優先：更晚發生的交叉訊號優先
    # 實務上，降低兩者分數的影響
    score_KD = score_KD × 0.6
    score_MACD = score_MACD × 0.6
    # 同時降低信心值
    信心值 = 信心值 × 0.7
```

**衝突場景 3：價格位置 vs 動能方向矛盾**

```
# 日內高檔 (+70以上) 但漲跌幅動能仍在加速 (+50以上)
if score_日內位置 > 70 and score_漲跌幅動能 > 50:
    # 動能優先於位置，視為「強勢突破」
    # 不做調整，但信心值小幅下修
    信心值 = 信心值 × 0.9
```

## 4. 自動驗證機制

### 4.1 驗證時機

推論完成後啟動 **62 秒** 倒數計時器，到時自動擷取最新成交價與推論當下的成交價進行比對。

**為何選擇 62 秒**：
- 符合台股盤中約 1 分鐘的短線波動週期
- 足夠短以反映即時分析的準確度
- 比 60 秒略長，避開整分鐘可能的集中競價雜訊

### 4.2 驗證規則

```
推論價格 = 推論當下的成交價
驗證價格 = 62秒後的最新成交價
價格變化 = 驗證價格 - 推論價格
變化率 = 價格變化 / 推論價格 × 100%
```

**判定標準**：

| 推論方向 | 價格變化 | 判定 |
|---------|---------|------|
| 看漲（總分 ≥ +5） | 變化率 > +0.02% | 成功 |
| 看漲（總分 ≥ +5） | 變化率 < -0.02% | 失敗 |
| 看漲（總分 ≥ +5） | -0.02% ≤ 變化率 ≤ +0.02% | 平盤（不計入） |
| 看跌（總分 ≤ -5） | 變化率 < -0.02% | 成功 |
| 看跌（總分 ≤ -5） | 變化率 > +0.02% | 失敗 |
| 看跌（總分 ≤ -5） | -0.02% ≤ 變化率 ≤ +0.02% | 平盤（不計入） |
| 中性（-4 ≤ 總分 ≤ +4） | -0.05% ≤ 變化率 ≤ +0.05% | 成功 |
| 中性（-4 ≤ 總分 ≤ +4） | \|變化率\| > 0.05% | 失敗 |

**平盤門檻 ±0.02% 的設計理由**：
- 台股最小跳動（tick）在不同價格區間為 0.01~5.00 元
- 以 100 元股票為例，1 個 tick = 0.1 元 = 0.1%，遠大於 0.02%
- 0.02% 對低價股（如 20 元）= 0.004 元，約為半個 tick 內的變化
- 實務上建議依股價動態調整：`平盤門檻 = max(0.02%, 最小tick / 目前價 × 50%)`

### 4.3 成功率統計

```
成功次數 = count(判定 == "成功")
失敗次數 = count(判定 == "失敗")
平盤次數 = count(判定 == "平盤")
有效推論次數 = 成功次數 + 失敗次數

成功率 = 成功次數 / 有效推論次數 × 100%
```

**統計維度**：

| 維度 | 說明 |
|------|------|
| 整體成功率 | 所有推論的成功率 |
| 依方向 | 看漲推論成功率 / 看跌推論成功率 / 中性推論成功率 |
| 依信心值 | 高信心(>70%)成功率 / 中信心(40-70%)成功率 / 低信心(<40%)成功率 |
| 依時段 | 開盤期(09:00-09:30) / 盤中(09:30-12:00) / 尾盤(12:00-13:30) |
| 依個股 | 各個股的個別成功率 |
| 滾動統計 | 最近 20 次 / 50 次 / 100 次的滾動成功率 |

### 4.4 權重持續優化機制

**目標**：根據歷史驗證結果，動態調整 9 大訊號的權重，使成功率持續提升。

**方法一：簡單正確率加權**

```
每日收盤後計算：

for each 訊號 i:
    正確次數_i = count(訊號i的方向 == 實際方向)
    訊號正確率_i = 正確次數_i / 有效次數

# 重新計算權重（以正確率為基礎）
sum_正確率 = Σ(訊號正確率_i)
for each 訊號 i:
    新權重_i = 訊號正確率_i / sum_正確率

# 平滑更新（避免劇烈變動）
最終權重_i = 0.7 × 原權重_i + 0.3 × 新權重_i

# 確保每個訊號權重在合理範圍
最終權重_i = clamp(最終權重_i, 0.03, 0.25)   # 最低 3%，最高 25%

# 正規化使總權重 = 100%
最終權重_i = 最終權重_i / Σ(最終權重_j)
```

**方法二：貢獻度分析加權（進階）**

```
# 計算每個訊號對成功推論的貢獻度
for each 成功推論:
    total_weighted = 加權總分
    for each 訊號 i:
        contribution_i = (score_i × weight_i) / total_weighted
        # 記錄每個訊號在成功推論中的貢獻度

# 統計每個訊號的平均正向貢獻度
avg_contribution_i = mean(contribution_i for all 成功推論)

# 以貢獻度重新分配權重
# 類似方法一，做平滑更新
```

**更新頻率**：
- 累積 50 次有效推論後開始第一次優化
- 之後每 100 次有效推論更新一次
- 保留所有歷史權重版本以便回溯比較
